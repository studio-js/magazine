Philosophy / Body

Defaults for Eye Data

Optic ID, system-level camera and sensor processing, and the principle that eye input is not shared are not minor notes. In a device controlled by gaze, default ethics become bodily boundaries.

PublishedApr 08, 2026 LocationCupertino Reading10 min read TagsApple / Privacy / Body
01 Invisible Boundaries

The user's gaze is an operating signal and also an interior.

As the body becomes interface, ethics begin with invisible defaults.

Invisible Boundaries

Apple says Vision Pro's eye input is not shared with Apple, third-party apps, or websites, and that only final selections are transmitted. It also says camera and sensor data are processed at the system level, so individual apps do not need to see the surroundings.

This language is not an appendix to feature description but close to the product's philosophical center. When gaze and hands become input devices, the user's body is both interface and an interior to be protected.

In a spatial interface, privacy is not a settings item behind the screen. Where a user looks, what they did not select, and what a lingering gaze might mean are all connected to the interior of the body.

Eye input is convenient, but even hesitation before selection can become data. A good spatial device should therefore be judged not only by what a user presses, but by how little bodily signal remains before the press.

Design That Leaves Only Selection

A good privacy default does not make safety depend on the user reviewing every setting. The safer direction should already be set, so work and viewing can continue on top of it.

Good privacy design must keep working when the user stops thinking about it.

The ethics of spatial computing are judged less by the scale of immersion than by how little of the user's body is exposed. As the eye becomes cursor, the person behind the cursor must be protected more quietly.

A good default decides direction before educating the user. This is where Vision Pro's privacy language becomes important: bodily data should be protected inside the product's first judgment without requiring the user to defend it each time.

Protection should therefore be an attitude at the moment the product first turns on, not an option that works after a user finds a menu. Taste in spatial computing is decided less by how many senses are read than by how little of what is read is externalized.
